# ADR-002: Выбор оркестратора

## Статус

Принято.

## Контекст

Нужно управлять batch ETL/ELT задачами, видеть статусы выполнения и логи.

## Решение

Выбран Prefect, потому что его удобно запускать локально в Docker, а flow можно описывать обычным Python-кодом.

## Последствия

Пайплайны размещены в каталоге `pipelines`, а Prefect UI используется для observability.
